OVERVIEW

Raptor is the RDF Parser Toolkit for Redland. It is designed to integrate with Redland closely and also work entirely standalone, in which case it will use simple internal routines to perform the functions.

This library is Alpha quality. See the todo list for more information. Changes can be found in the NEWS file or more detailed changes in the ChangeLog.

Features

  • An RDF/XML Parser written in C designed to integrate well with Redland
  • Can extract RDF content embedded in XML (such as XHTML)
  • Handles rdf:resource / resource attributes
  • Optionally can interpret container elements (only) as typedNode after the proposed interpretation by Brian McBride and myself.
  • Parsing tracks the formal grammar productions closely - this can be followed when debugging.
  • Uses expat and/or (GNOME) libxml XML parsers as available or required
  • Has hooks for new parseType-s and container types
  • Provides features to select options at run time.
  • (Perl and Python interfaces when used via Redland)
  • No memory leaks
  • Fast

WARNING This is alpha quality software - the APIs will change, it is not feature complete and it needs lots of testing and conformance checking.
